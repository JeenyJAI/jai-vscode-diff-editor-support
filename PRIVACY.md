# Privacy Policy

## Telemetry

JAI: Diff Editor collects minimal, anonymous telemetry to understand how the extension is used and to prioritize development efforts.

### What We Collect

**Usage events (triggered by user actions):**

- `panel/opened` — Diff Editor panel was opened (new or revealed)
- `command/preview` — Preview button clicked (number of diff blocks, duration)
- `command/apply` — Apply button clicked (number of hunks, number of files, duration)
- `command/openDiff` — Diff Viewer opened
- `mcp/enabled` — MCP Server started

**Automatic properties (added by VS Code telemetry library):**

- Extension version
- VS Code version
- Operating system and version
- Anonymous machine ID (generated by VS Code)
- Session ID
- Country/region (determined by Azure from IP address)

### What We Do NOT Collect

- File contents, patch contents, or diffs
- File paths (absolute or relative)
- File names
- Usernames or any personally identifiable information (PII)
- Workspace data (folder names, project structure)
- IP addresses (used for geolocation only, not stored)

### Where Data Is Sent

Telemetry is sent to [Azure Application Insights](https://learn.microsoft.com/en-us/azure/azure-monitor/app/app-insights-overview), a Microsoft service for application monitoring.

### How to Disable Telemetry

Set the following in your VS Code settings:

```json
{
  "telemetry.telemetryLevel": "off"
}
```

This disables telemetry for all extensions that respect the VS Code telemetry setting. No data will be sent when this setting is set to `off`.

You can also use `"error"` or `"crash"` levels for partial telemetry — since this extension currently only sends usage events (not errors or crashes), these levels will also result in no data being sent.
